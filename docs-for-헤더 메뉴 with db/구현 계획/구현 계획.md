# [구현 계획] 헤더 메뉴 DB화 및 동적 렌더링

> `beauty-book-hair` 프로젝트의 검증된 패턴(Flat Storage → Tree Transformation → Recursive Rendering)을 참고하여 `towercrane-for-uiux`에 최적화된 방식으로 구현합니다.

## 1. 개요 및 기대 효과

### 왜 하는가?
- **유연한 구조**: 하드코딩된 메뉴는 구조 변경 시마다 프론트엔드 코드 수정 및 배포가 필요함.
- **권한 제어**: 사용자 역할(Role)에 따른 메뉴 노출/비노출을 DB 레벨에서 관리 가능.
- **N차 메뉴 지원**: 무제한 depth의 계층형 메뉴 구성을 코드 변경 없이 지원.
- **관리 효율**: 추후 관리자 UI를 통해 비개발자도 사이트 구조를 변경할 수 있는 기반 마련.

### 핵심 원칙 (beauty-book-hair 계승)
1. **저장 및 전송은 플랫(Flat)**: DB 테이블과 API 응답은 부모 ID를 가진 평면 리스트 형태.
2. **출력 및 관리는 트리(Tree)**: 프론트엔드에서 `buildTree` 유틸을 통해 계층 구조로 변환 후 재귀 렌더링.
3. **단일 진실 공급원(Single Source of Truth)**: 메뉴 구성의 모든 정보는 DB에서 관리.

## 2. 상세 구현 계획

### Step 1: 백엔드 기반 구축 (NestJS + Drizzle)
- [ ] **DB 스키마 설계**: `menus` 테이블 생성
    - `id`, `parent_id` (자기 참조), `name`, `section_id` (SPA 라우팅용, ex: 'prototype', 'users'), `icon`, `display_order`, `is_visible`, `required_role` 등.
- [ ] **Drizzle Entity & Repository**: 엔티티 정의 및 CRUD 로직 구현.
- [ ] **Menu API 구현**:
    - `GET /api/menus`: 전체 메뉴 리스트 반환 (Flat 형태).
    - `POST/PATCH/DELETE /api/menus`: 관리자용 CRUD API.
- [ ] **Menu Seeder**: 초기 기본 메뉴 데이터 세팅.

### Step 2: 프론트엔드 연동 (React + TanStack Query)
- [ ] **Menu API 연동**: `useQuery`를 이용한 메뉴 데이터 페칭.
- [ ] **Tree 변환 유틸**: `buildTree(flatList)` 함수 구현 (부모-자식 관계 재구성).
- [ ] **재귀적 Header 렌더링**:
    - 기존 하드코딩된 `Header.tsx`를 동적 렌더링 방식으로 전환.
    - 서브메뉴(Dropdown)가 있는 경우 재귀적으로 컴포넌트 호출.

### Step 3: 관리자 UI (선택 사항 - v2)
- [ ] **메뉴 관리 페이지**: 트리 뷰 기반의 메뉴 관리 인터페이스.
- [ ] **Drag & Drop (dnd-kit)**: 메뉴 순서 변경 및 계층 이동 기능.

## 3. 기술적 결정 사항

| 구분 | 결정 사항 | 비고 |
|------|----------|------|
| **데이터 구조** | Flat List (Parent-ID) | DB 부하 최소화 및 전송 효율 |
| **라우팅 방식** | `section_id` 기반 상태 제어 | Next.js(`path` 기반)과 달리 Zustand(`activeSection`)를 사용하므로 DB 필드를 `section_id`로 조정 |
| **트리 변환** | Client-side (`buildTree`) | 서버 연산 부담 감소, 프론트 유연성 확보 |
| **상태 관리** | TanStack Query | 캐싱 및 자동 갱신 활용 |
| **스타일링** | Tailwind Semantic Tokens | `AGENTS.md` 규칙 준수 (`text-text-primary` 등) |
