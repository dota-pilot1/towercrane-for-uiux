# [참고 파일] beauty-book-hair 메뉴 DB화 구현 사례

> 이 문서는 `beauty-book-hair` 프로젝트에서 메뉴를 DB화하여 동적으로 관리하고 렌더링했던 핵심 코드들을 모아둔 것입니다. `towercrane-for-uiux` 구현 시 구조와 로직의 참고용으로 사용합니다.

## 1. 백엔드 (Java/Spring 기준)

### 1-1. Menu 엔티티 (`Menu.java`)
- `parent_id`를 통한 자기 참조(Self-referencing) 구조.
- `displayOrder`, `visible`, `requiredRole` 등의 필드로 출력 제어.

```java
@Entity
@Table(name = "menus", uniqueConstraints = @UniqueConstraint(columnNames = "code"))
public class Menu {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false, length = 100, unique = true)
    private String code;

    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "parent_id")
    private Menu parent;

    @Column(nullable = false, length = 100)
    private String label;

    private String path;
    private String icon;
    private String requiredRole;
    private boolean visible = true;
    private int displayOrder = 0;
}
```

### 1-2. 초기 데이터 시더 (`MenuSeeder.java`)
- 애플리케이션 기동 시 기본 메뉴를 자동으로 생성.
- 계층 구조(ADMIN > ADMIN_USERS 등)를 설정하는 방식 참고.

```java
new MenuDef("ADMIN", null, "관리", "nav.admin", null, "Settings", ROLE_ADMIN, 10),
```

## 2. 프론트엔드 (React/Next.js 기준)

### 2-1. 타입 정의 및 API (`types.ts`, `menuApi.ts`)

### 2-2. 트리 변환 유틸 및 필터링 (`buildTree`)
- `beauty-book-hair`의 `Header.tsx` 내부에 구현된 핵심 로직.
- **Flat → Tree** 변환과 동시에 **권한 필터링**을 수행.

```typescript
function buildTree(flat: MenuRecord[], userRole: string | null): MenuItem[] {
  // 1. 가시성 및 권한 필터링
  const visible = flat.filter(
    (m) => m.visible && (!m.requiredRole || m.requiredRole === userRole)
  );

  // 2. ID 맵 생성
  const map = new Map<number, MenuItem>();
  visible.forEach((m) => map.set(m.id, { ...m, children: [] }));

  // 3. 계층 구조 조립
  const roots: MenuItem[] = [];
  map.forEach((item) => {
    if (item.parentId === null) {
      roots.push(item);
    } else {
      map.get(item.parentId)?.children.push(item);
    }
  });

  // 4. 순서 정렬
  const sort = (items: MenuItem[]) => items.sort((a, b) => a.displayOrder - b.displayOrder);
  map.forEach((item) => sort(item.children));
  return sort(roots);
}
```

### 2-3. 헤더 재귀 렌더링 패턴
- `NavItem`이 자식이 있으면 `Dropdown`을, 없으면 `NavLink`를 렌더링하는 구조.

```typescript
function NavItem({ item }: { item: MenuItem }) {
  if (item.children.length > 0) {
    return <DropdownMenu item={item} />;
  }
  return <NavLink href={item.path ?? "#"}>{item.label}</NavLink>;
}
```

## 3. 구현 시 유의사항
- **아이콘 매핑**: DB에는 `Lucide` 아이콘 이름만 문자열로 저장하고, 프론트에서 동적으로 컴포넌트를 매핑하는 로직이 필요합니다.
- **스타일링**: `beauty-book-hair`는 Vanilla CSS/Tailwind를 혼용했으나, `towercrane-for-uiux`에서는 `AGENTS.md`에 정의된 **Semantic Tokens**를 엄격히 준수해야 합니다.
- **백엔드 차이**: 참고 프로젝트는 Java/Spring이지만 현재 프로젝트는 **NestJS + Drizzle**이므로 데이터베이스 모델링 및 API 구현 시 문법적 차이에 주의합니다.
