## 1. 사전 준비 (로컬에서 할 일)

   ```bash
   # SSH 접속
   ssh -i hibot-d-server-key.pem ubuntu@54.180.215.129
   ```

## 2. 기존 프로젝트 정리 (내리기)

기존에 실행 중이던 프로젝트(hibot-docu)의 PM2 프로세스를 중지하고 삭제합니다.

```bash
# PM2에서 실행 중인 프로세스 목록 확인
pm2 list

# 기존 앱 중지 (백엔드, 프론트엔드 모두)
pm2 stop hibot-docu-back
pm2 stop hibot-docu-front

# PM2 리스트에서 제거
pm2 delete hibot-docu-back
pm2 delete hibot-docu-front

# 서버 재부팅 시 반영을 위한 PM2 상태 저장
pm2 save
```

## 3. 새 프로젝트 클론 및 세팅

서버 홈 디렉토리(`~/`)에 새 프로젝트를 가져옵니다.

```bash
cd ~
git clone <새_프로젝트_Git_주소> towercrane-project
cd towercrane-project
```

## 4. 백엔드 (NestJS) 배포 설정

1. **디렉토리 이동 및 설치**
   ```bash
   cd ~/towercrane-project/towercrane-for-uiux-server
   npm install
   ```
2. **환경변수(.env) 설정**
   사전에 필요한 `.env` 파일을 서버에 생성합니다. (기존 DB 정보 등을 활용)
   ```bash
   nano .env
   ```
3. **DB 스키마 마이그레이션 적용**
   ```bash
   # Drizzle을 사용하는 경우
   npx drizzle-kit push
   ```
4. **빌드 및 PM2 실행**
   ```bash
   npm run build
   pm2 start npm --name "towercrane-back" -- run start:prod
   ```

## 5. 프론트엔드 (React + Vite) 배포 설정

Vite는 정적 빌드(`dist/` 폴더)를 생성하므로 별도 Node 서버 없이 S3 정적 호스팅 또는 Nginx로 서빙합니다.

1. **디렉토리 이동 및 설치**
   ```bash
   cd ~/towercrane-project/towercrane-for-uiux-front
   npm install
   ```
2. **환경변수 설정**
   백엔드의 실서버 주소를 가리키는 환경변수를 설정합니다. (`.env.production` 생성)
   Vite 환경변수는 반드시 `VITE_` 접두사를 사용해야 합니다.
   ```bash
   nano .env.production
   # 내용 예시:
   # VITE_API_BASE_URL=http://54.180.215.129:<백엔드포트>/api
   ```
3. **빌드**
   ```bash
   npm run build
   # 빌드 결과물은 dist/ 폴더에 생성됩니다 (Next.js의 out/ 아님)
   ```
4. **S3에 업로드 (정적 호스팅)**
   ```bash
   aws s3 sync dist/ s3://내_S3_버킷명 --delete
   ```
   또는 **EC2 서버에서 Nginx로 서빙** (S3 없이 EC2에서 직접 호스팅 시)
   ```bash
   # dist/ 를 Nginx 서빙 경로로 복사
   sudo cp -r dist/* /var/www/html/
   sudo systemctl restart nginx
   ```
   > Vite는 SSR이 없으므로 PM2로 프론트엔드 서버를 따로 띄울 필요가 없습니다.
