## 3. 백엔드 설정용 환경 변수 (.env)

백엔드 서버를 띄울 때 필요한 3rd-party API 키와 AWS 파일 업로드 스토리지 접근 등을 위한 기존 환경 변수들입니다. 새로 올리는 프로젝트에서도 비슷한 기능이 들어간다면 이를 복사하여 서버/로컬 `.env`에 설정하세요.

### AWS S3 Configuration (이미지 등 파일 업로드 서비스용)
> (주의: 프론트엔드 정적 호스팅 배포용 계정 키와 별도입니다)
```env
AWS_ACCESS_KEY_ID=<AWS_ACCESS_KEY_ID>
AWS_SECRET_ACCESS_KEY=<AWS_SECRET_ACCESS_KEY>
AWS_S3_BUCKET_NAME=hibot-docu
AWS_S3_REGION=ap-northeast-2
```

### AI Configuration (Gemini / OpenAI API)
```env
GEMINI_API_KEY=<GEMINI_API_KEY>
OPENAI_API_KEY=<OPENAI_API_KEY>
```

### DB 및 기타 설정 정보 (권장 예시)
서버 EC2 내부에 동일하게 구축된 Postgres를 사용한다면 기존 정보를 그대로 사용하세요.
```env
DB_HOST=localhost
DB_PORT=5433
DB_USERNAME=postgres
DB_PASSWORD=postgres
DB_DATABASE=hibot_docu  # (새 프로젝트라면 데이터베이스 스키마 생성을 위해 이름 변경을 권장합니다, 예: towercrane_db)

JWT_SECRET=your-secret-key-change-in-production
```

## 4. 백엔드 서버 URL (프론트엔드 `.env.production` 용)

새로 띄울 Towercrane 백엔드 서버가 로컬 포트 4001이나 Nginx/도메인 설정을 이용해 퍼블릭으로 뚫려있다면, 프론트엔드의 환경 변수에는 해당 주소나 직접 IP 주소를 지정하여 빌드해야 합니다.

*   (도메인이 없는 경우 예시) `NEXT_PUBLIC_API_URL=http://54.180.215.129:`<EC2_Port번호>
*   (기존 도메인 사용 시) `NEXT_PUBLIC_API_URL=https://api.hibot-docu.com`
