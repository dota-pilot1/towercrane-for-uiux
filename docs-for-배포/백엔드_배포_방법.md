# 백엔드 배포 방법

> 대상: `towercrane-for-uiux-server`  
> 방식: EC2 접속 → `main` pull → NestJS build → PM2 restart

## 배포 정보

| 항목 | 값 |
|---|---|
| EC2 Public IP | `54.180.215.129` |
| SSH User | `ubuntu` |
| SSH Key | `docs-for-배포/hibot-d-server-key.pem` |
| API 도메인 | `https://api.hibot-docu.com/api` |
| 앱 포트 | `4001` |
| PM2 앱 이름 | `towercrane-back` |
| DB | SQLite |
| 기본 DB 파일 | `./data/towercrane-catalog.sqlite` |

## 언제 백엔드를 배포하나

- controller/service/module 변경
- DB schema 또는 초기 seed/migration 로직 변경
- 프론트가 새 API나 새 response field에 의존
- 서버 `.env` 변경
- 파일 업로드, AI API, 인증 등 서버 런타임 설정 변경

프론트가 새 백엔드 동작에 의존하면 백엔드를 먼저 배포하고 API 응답을 확인한 뒤 프론트를 배포합니다.

## SSH 접속

```bash
cd docs-for-배포
chmod 400 hibot-d-server-key.pem
ssh -i hibot-d-server-key.pem ubuntu@54.180.215.129
```

## 최초 배포

```bash
cd ~
git clone https://github.com/dota-pilot1/towercrane-for-uiux.git towercrane
cd ~/towercrane/towercrane-for-uiux-server

cat > .env << 'EOF'
PORT=4001
DATABASE_FILE=./data/towercrane-catalog.sqlite
AWS_ACCESS_KEY_ID=<서버에서만_관리>
AWS_SECRET_ACCESS_KEY=<서버에서만_관리>
AWS_S3_BUCKET_NAME=hibot-docu
AWS_S3_REGION=ap-northeast-2
GEMINI_API_KEY=
OPENAI_API_KEY=
EOF

mkdir -p data
npm install
npm run build
pm2 start dist/main.js --name towercrane-back
pm2 save
```

## 코드 업데이트 재배포

서버 배포는 원격 `main` 기준입니다. 로컬 변경이 아직 push되지 않았다면 먼저 GitHub에 반영합니다.

권장 명령:

```bash
./scripts/deploy-backend.sh
```

스크립트가 수행하는 안전장치:

- EC2에서 `/tmp/towercrane-backend-deploy.lock`을 잡아 동시 배포 방지
- 배포 직전 커밋을 `/tmp/towercrane-backend-rollback-commit`에 기록
- SQLite DB 파일을 `data/backups/`에 백업
- `pnpm-lock.yaml`이 있으면 `pnpm install --frozen-lockfile` 우선 사용
- PM2 재시작 후 `https://api.hibot-docu.com/api/menus` 헬스체크
- 헬스체크 실패 시 직전 커밋으로 checkout/build/restart 롤백 시도

수동 재배포가 필요할 때만 아래 절차를 사용합니다.

```bash
cd ~/towercrane
git pull origin main

cd towercrane-for-uiux-server
npm install
npm run build
pm2 restart towercrane-back
pm2 save
```

## DB 변경이 포함된 경우

현재 서버는 앱 시작 시 필요한 SQLite 테이블/레거시 마이그레이션을 `DatabaseService`에서 보정합니다. 그래도 DB 변경 배포 전에는 반드시 DB 파일을 백업합니다.

```bash
cd ~/towercrane/towercrane-for-uiux-server
mkdir -p data/backups
cp data/towercrane-catalog.sqlite "data/backups/towercrane-catalog.$(date +%Y%m%d-%H%M%S).sqlite"
```

Drizzle SQL migration을 명시적으로 생성/적용하는 운영 절차가 생기면 이 문서에 별도 단계로 추가합니다.

## PM2 확인

```bash
pm2 list
pm2 logs towercrane-back --lines 80
pm2 describe towercrane-back
```

정상 재시작 후 `status`가 `online`인지 확인합니다.

## nginx 확인

백엔드는 `localhost:4001`로 실행되고 nginx가 `api.hibot-docu.com`을 프록시합니다.

```bash
sudo nginx -t
sudo systemctl reload nginx
```

nginx 설정 파일은 `/etc/nginx/sites-enabled/`에서 확인합니다.

## 배포 후 확인

```bash
curl -fsS https://api.hibot-docu.com/api/menus | head -c 500
curl -fsSI https://api.hibot-docu.com/api/menus | sed -n '1,20p'
```

문제가 있으면 PM2 로그와 nginx 로그를 함께 봅니다.

```bash
pm2 logs towercrane-back --lines 120
sudo tail -n 120 /var/log/nginx/error.log
```

## 롤백

스크립트 배포 중 헬스체크가 실패하면 자동 롤백을 시도합니다. 수동 롤백은 아래 절차를 사용합니다.

```bash
cd ~/towercrane
git log --oneline -5
git checkout <이전_정상_커밋>

cd towercrane-for-uiux-server
npm install
npm run build
pm2 restart towercrane-back
pm2 save
```

롤백 후 다시 `main`으로 복귀해야 할 때는 `git checkout main && git pull origin main`을 실행합니다.

## 주의

- `.env`와 SQLite DB 파일은 git에 포함하지 않습니다.
- DB 파일 삭제는 운영 데이터 유실로 이어집니다.
- `pm2 save`는 서버 재부팅 후 현재 프로세스 구성을 복구하기 위해 필요합니다.
