# 프로젝트 이미지 업로드(S3 Presigned URL) 구현 가이드

## 1. 백엔드 구현 상태 평가
이 프로젝트의 백엔드 이미지 업로드 기능은 **공통 기능(Common Module)으로서 매우 정석적이고 우수하게 구현**되어 있습니다.

### 👍 잘된 점 (Best Practices)
- **Presigned URL 패턴 사용**: 클라이언트가 서버를 거치지 않고 S3로 직접 파일을 업로드하므로 서버의 트래픽과 메모리 부하를 방지합니다.
- **안전한 파일명 처리**: UUID를 사용하여 파일명 충돌을 방지하고 `safeName` 치환 로직을 두었습니다.
- **모듈화**: `UploadModule`로 기능이 깔끔하게 분리되어 있어 다른 프로젝트에 가져다 쓰기 (Drop-in) 좋은 구조입니다.

### 💡 개선하면 더 완벽할 점
- 현재 `upload.service.ts`의 파일 저장 경로가 `towercrane-docu/` 로 하드코딩 되어 있습니다. 이를 클라이언트로부터 `prefix`나 `folder` 값을 인자로 받게 하면, 도메인별(유저 프로필, 리뷰 이미지 등)로 경로를 나누어 더 완벽한 범용 모듈로 사용할 수 있습니다.

---

## 2. 다른 프로젝트에서 참고할 파일 구조
이 패턴을 다른 프로젝트에 똑같이 적용하고 싶다면 아래 4개의 핵심 파일만 참조하여 구성하면 됩니다.

### [Backend: NestJS 기반]
1. `src/upload/upload.module.ts`: 의존성 주입을 위한 모듈 정의
2. `src/upload/upload.controller.ts`: `/upload/presign` 엔드포인트를 노출하여 클라이언트 요청을 받는 진입점 담당
3. `src/upload/upload.service.ts`: 실제 `@aws-sdk/client-s3` 패키지를 사용하여 Presigned URL을 발급하는 핵심 비즈니스 로직 담당

### [Frontend: React/TypeScript 기반]
4. `src/shared/api/upload.ts`: 프론트엔드에서 서버에 Presigned URL을 요청하고, 그 URL을 통해 S3로 파일을 즉시 `PUT` 하는 과정을 묶어놓은 공통 API 래퍼(Wrapper) 함수

## 3. 구현 방식 및 동작 흐름 (How to Implement)

### Flow (업로드 시퀀스)
1. **Frontend**: 사용자가 파일을 선택하면 서버 API(`/upload/presign`)를 호출하여 업로드할 권한(Presigned URL)을 요청합니다. (이때 파일명과 Content-Type을 함께 보냅니다.)
2. **Backend**: AWS S3에 접근하여 "잠시 동안(예: 300초) 이 파일 이름으로 업로드 할 수 있는 임시 URL"을 생성해서 반환해줍니다.
3. **Frontend**: 발급받은 임시 URL(Presigned URL)을 사용해 `fetch(..., { method: 'PUT', body: file })` 방식으로 AWS S3 버킷에 직접 파일을 던집니다.
4. **Frontend**: S3 업로드가 성공하면 S3의 Public URL을 결과로 받아서 폼 데이터에 연결하거나 화면에 렌더링 할 수 있습니다.

### 코드 스니펫 핵심 참고 포인트

#### Backend (`upload.service.ts` 요약)
```typescript
import { S3Client, PutObjectCommand } from '@aws-sdk/client-s3';
import { getSignedUrl } from '@aws-sdk/s3-request-presigner';

// 1. S3 Client 설정
const s3Client = new S3Client({
  region: process.env.AWS_S3_REGION,
  credentials: { accessKeyId, secretAccessKey },
});

// 2. Presigned URL 생성 로직
const command = new PutObjectCommand({
  Bucket: process.env.AWS_S3_BUCKET_NAME,
  Key: `my-folder/${uuid}-${filename}`,
  ContentType: contentType,
});

const presignedUrl = await getSignedUrl(s3Client, command, { expiresIn: 300 });
// 클라이언트에 presignedUrl과 최종적으로 사용할 publicUrl을 리턴
return { presignedUrl, publicUrl };
```

#### Frontend (`upload.ts` 요약)
```typescript
export async function uploadFile(file: File): Promise<string> {
  // 1. 서버에 Presigned URL 요청
  const { presignedUrl, publicUrl } = await api.post('/upload/presign', {
    filename: file.name,
    contentType: file.type,
  });

  // 2. S3로 리소스 직접 PUT (form-data가 아닌 원시 바이너리 형태)
  const uploadResponse = await fetch(presignedUrl, {
    method: 'PUT',
    body: file,
    headers: {
      'Content-Type': file.type, // S3에서 권한 처리시 타입을 검사하므로 반드시 맞추어야 함
    },
  });

  if (!uploadResponse.ok) throw new Error('업로드 실패');
  
  // 3. 업로드 성공 시 이미지 URL 리턴
  return publicUrl;
}
```
