# 테마 색상 설정

towercrane-for-uiux-front 프로젝트의 색상/테마 체계 정리 문서입니다.
이 문서의 목표: **raw Tailwind 팔레트 금지, semantic token 기반 일관 UI**.

---

## 1. 구조 한눈에 보기

```
:root (CSS variables)       ← 테마별 값 정의
    ↓
@theme (Tailwind token)     ← CSS var 를 Tailwind 유틸로 노출
    ↓
@layer utilities (ui-*)     ← 프리셋 조합 (panel / icon button / input)
    ↓
컴포넌트 (shared/ui/*)       ← 위 토큰/유틸로 스타일링
```

---

## 2. CSS 변수 계층 (src/index.css `:root`)

| 그룹 | 변수 | 용도 |
|---|---|---|
| 브랜드 | `--brand-500` | 포인트 색 (초록/파랑 등 테마별 변경) |
| 브랜드 | `--brand-glass` | 반투명 포인트 배경 (액티브 탭, 뱃지) |
| 브랜드 | `--brand-border` | 포인트 보더 |
| 브랜드 | `--brand-glow` | 배경 글로우 radial gradient |
| 앱 | `--app-bg` | 전역 배경 (그라디언트 포함) |
| 앱 | `--app-fg` | 전역 기본 글자색 |
| 서피스 | `--surface-raised` | 카드/패널 배경 |
| 서피스 | `--surface-muted` | 은은한 배경 (hover, 섹션 구분) |
| 서피스 | `--surface-strong` | 강조 배경 (드롭다운, 어두운 패널) |
| 서피스 | `--surface-border` | 보더 기본 |
| 서피스 | `--surface-border-soft` | 보더 연한 버전 |
| 서피스 | `--surface-shadow` | 박스 섀도우 |
| 텍스트 | `--text-primary` | 본문 기본 |
| 텍스트 | `--text-secondary` | 보조 설명 |
| 텍스트 | `--text-muted` | 힌트 / placeholder |
| 입력 | `--input-bg` / `--input-border` / `--input-placeholder` | 입력 필드 |
| 인터랙티브 | `--interactive-border` | 버튼/탭 기본 보더 |

### 테마 스위치

`document.documentElement` 의 `data-theme` attribute로 테마 전환:
- 기본: dark (emerald brand)
- `data-theme="blue"` / `"violet"` / `"rose"` / `"amber"`: brand 색만 교체
- `data-theme="light"`: 배경/서피스/텍스트 전부 light 재정의

## 3. Tailwind Semantic Token (src/index.css `@theme`)

CSS 변수를 Tailwind 유틸로 자동 생성하는 매핑:

```css
@theme {
  --color-brand-primary: var(--brand-500);
  --color-brand-glass: var(--brand-glass);
  --color-brand-border: var(--brand-border);
  --color-text-primary: var(--text-primary);
  --color-text-secondary: var(--text-secondary);
  --color-text-muted: var(--text-muted);
  --color-surface-raised: var(--surface-raised);
  --color-surface-muted: var(--surface-muted);
  --color-surface-strong: var(--surface-strong);
  --color-surface-border: var(--surface-border);
  --color-surface-border-soft: var(--surface-border-soft);
}
```

### 자동 생성되는 유틸 (예시)

| Tailwind 유틸 | 효과 |
|---|---|
| `text-text-primary` | 본문 색 (테마 대응) |
| `text-text-secondary` | 보조 색 |
| `text-text-muted` | 힌트 색 |
| `text-brand-primary` | 포인트 색 |
| `bg-brand-glass` | 반투명 포인트 배경 |
| `bg-surface-muted` | 은은한 배경 |
| `bg-surface-raised` | 카드 배경 |
| `bg-surface-strong` | 강조 배경 |
| `border-brand-border` | 포인트 보더 |
| `border-surface-border` | 기본 보더 |
| `border-surface-border-soft` | 연한 보더 |

## 4. ui-\* 유틸 클래스 (src/index.css `@layer utilities`)

자주 쓰는 조합을 프리셋으로 정의:

| 클래스 | 역할 |
|---|---|
| `glass-panel` | 라운드 카드 (보더 + 배경 + 블러 + 섀도우) |
| `ui-overlay` | 모달 오버레이 (반투명 어두운 backdrop) |
| `ui-surface-muted` | surface-muted 배경 + border |
| `ui-input` | 입력 필드 (배경 + 보더 + 텍스트 색) |
| `ui-text-primary` / `ui-text-secondary` / `ui-text-muted` | 텍스트 색 |
| `ui-panel` | 내부 패널 (보더 + surface-muted 배경) |
| `ui-panel-soft` | 연한 보더 버전 |
| `ui-icon-button` | 기본 아이콘 버튼 |
| `ui-icon-button-brand` | 포인트 색 아이콘 버튼 |
| `ui-icon-button-danger` | 위험(삭제) 아이콘 버튼 |

## 5. 새 UI 작성 체크리스트

### ✅ 해야 할 것
1. `shared/ui/`에 이미 있는 컴포넌트인가? → 재사용 (`Card`, `Button`, `Input`, `Dialog`)
2. Semantic token으로 표현 가능한가? → `text-text-secondary`, `bg-surface-muted` 등
3. `ui-*` 유틸로 커버되는가? → `ui-panel-soft`, `ui-icon-button-brand` 등
4. 그래도 안 되면 CSS 변수 직접 (`bg-[var(--surface-muted)]`)

### ❌ 절대 쓰면 안 되는 것

| 금지 | 대체 |
|---|---|
| `text-white` | `text-text-primary` |
| `text-slate-400` | `text-text-secondary` |
| `text-slate-500` | `text-text-muted` |
| `text-slate-300` | `text-text-secondary` |
| `text-slate-200` | `text-text-primary` |
| `text-emerald-400` / `300` / `200` | `text-brand-primary` |
| `bg-white/10` | `bg-surface-muted` |
| `bg-slate-950/40` | `bg-surface-muted` |
| `bg-slate-900/*` | `bg-surface-muted` |
| `bg-emerald-500/10` | `bg-brand-glass` |
| `border-white/10` | `border-surface-border-soft` |
| `border-emerald-400` | `border-brand-border` |

### 예외 (raw 색상 허용되는 경우)
- 에러 표시용 rose/red: `text-rose-300`, `text-rose-400` 등은 유지
- Lexical 에디터 내부 (`shared/ui/lexical/*`): Lexical 엔진이 특정 클래스명을 키로 쓰므로 건드리지 않음
- `Button` primary variant: 브랜드 핵심 버튼은 고의로 고정 색 유지 (디자이너 확정 색)

## 6. 테마 추가 방법

새 브랜드 색 테마를 추가하려면:

```css
:root[data-theme="이름"] {
  --brand-500: #원하는색;
  --brand-glow: rgba(원하는색, 0.45);
  --brand-glass: rgba(원하는색, 0.1);
  --brand-border: rgba(원하는색, 0.2);
}
```

그리고 `ThemeSwitcher` 컴포넌트의 theme 리스트에 추가.

Light 테마처럼 전체 서피스까지 바꾸려면 위 변수들도 함께 재정의.

---

## 7. 반복 패턴은 컴포넌트로 승격

같은 "아이콘 박스 + 제목 + 설명" 이나 "active/inactive 리스트 항목" 같은 조합이 3번 이상 반복되면 `shared/ui/`로 뽑는다.

이유: 색상 결정이 한 곳에 모이면 테마 드리프트가 사라지고, 팔레트 변경 시 전 영역이 일관되게 움직인다.

---

## 8. 리팩토링 히스토리

- 초기: 페이지별로 `text-white`, `text-slate-*`, `bg-emerald-500/10` 같은 raw 팔레트 사용 → light 테마 전환 시 글씨/아이콘 안 보이는 문제 발생
- 1차: CSS 변수(`--text-*`, `--surface-*`) 도입 + `ui-*` 유틸 추가
- 2차: Tailwind `@theme` 확장으로 `text-text-*`, `bg-surface-*` 등 semantic 유틸 자동 생성
- 3차: 기존 raw 팔레트 300여 곳을 sed/perl 일괄 치환으로 semantic token으로 교체
- 4차: CLAUDE.md에 raw 팔레트 금지 규칙 등록, 재발 방지
